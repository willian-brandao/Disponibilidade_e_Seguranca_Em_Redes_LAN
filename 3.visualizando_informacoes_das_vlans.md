# Comandos de Verificação de VLAN - Manual de Consulta

## Verificar Todas as VLANs Configuradas
### Exibe todas as VLANs existentes no switch
```
display vlan
```
Explicação: Mostra todas as VLANs criadas no switch

Exibe a VLAN padrão do sistema (normalmente VLAN 1)

## Lista todas as VLANs configuradas (ex: VLAN 1, VLAN 2, VLAN 10)

## Verificar Portas de uma VLAN Específica

### Exibe as portas associadas a uma VLAN específica
```
display vlan vlan-id
```
Exemplo:
Verifica as portas na VLAN 2
```
display vlan 2
```
Explicação: Mostra todas as portas associadas à VLAN especificada

Indica o comportamento de tagging de cada porta

**Exemplo: GigabitEthernet1/0/1, GigabitEthernet1/0/3 e GigabitEthernet1/0/4 na VLAN 2.**

## Exemplo de Saída Esperada

## Informações Importantes
VLAN 1 é sempre a VLAN padrão do sistema

Portas listadas como Untagged removem a tag VLAN antes de encaminhar os quadros

Portas listadas como Tagged mantêm as tags VLAN nos quadros

O comando pode ser executado em qualquer visualização do switch

## Aplicação Prática
Use estes comandos para:

* Verificar a configuração atual de VLANs

* Identificar portas associadas a cada VLAN

* Validar configurações após mudanças

* Solucionar problemas de conectividade

# Comandos de Verificação de Interface - Manual de Consulta
## Verificar Informações de VLAN de uma Porta Específica

Exibe informações detalhadas de VLAN de uma interface específica
```
display interface interface-type interface-number
````
Explicação:

* Mostra a configuração completa de VLAN de uma porta específica

* Exibe o tipo de porta (acesso, tronco ou híbrida)

* Indica o PVID (VLAN padrão) da porta

## Exemplos de Uso
Verificar porta GigabitEthernet1/0/1:
```
display interface GigabitEthernet1/0/1
Exemplos de Saída Esperada
# Para Porta de Acesso:

Interface: GigabitEthernet1/0/1
Port link-type: access
PVID: 1
Tagged VLANs: -
Untagged VLANs: 1
Port status: up

```
Interpretação:

* Porta do tipo acesso

* PVID: 1 - VLAN padrão é VLAN 1

* Apenas tráfego untagged da VLAN 1

Para Porta Tronco ou Híbrida:
```
Interface: GigabitEthernet1/0/1
Port link-type: trunk/hybrid
PVID: 10
Tagged VLANs: 20,30,40
Untagged VLANs: 10,50
Port status: up
```
Interpretação:

* Porta do tipo trunk ou híbrida

* PVID: 10 - VLAN padrão é VLAN 10

* Tagged VLANs: Quadros mantêm tags VLAN (20,30,40)

* Untagged VLANs: Quadros têm tags removidas (10,50)

## Informações Importantes
O comando pode ser executado em qualquer modo de exibição

Para portas tronco e híbridas, mostra claramente:

* VLANs com quadros tagged (com marcação)

* VLANs com quadros untagged (sem marcação)

* O PVID indica a VLAN nativa para quadros sem tag

* Útil para validar configurações e solucionar problemas

## Aplicação Prática
### Use este comando para:

* Verificar o tipo de configuração de uma porta

* Validar o PVID configurado

* Confirmar VLANs tagged e untagged em portas trunk/híbridas

* Diagnosticar problemas de conectividade de VLAN

* Auditar configurações de rede
